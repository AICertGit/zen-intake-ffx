Our Solution

ZenIntake is an applicant-first portal that automates pre-screening while preserving human-in-the-loop governance.

1. Applicant Intake

Resident enters core application details in a guided form.

2. AI Normalization

Azure OpenAI optionally extracts structured fields from OCR text.

3. Identity Match

Fuzzy MCI matching links to existing golden records and reduces duplicates.

4. Employer Verify

Crustdata validates declared employer with resilient fallback behavior.

5. Decision Routing

Rules output AUTO_APPROVE, MANUAL_REVIEW, or NOT_ELIGIBLE.

How Client Matching Works

Identity Resolution
  • Input: applicant name + address
  • Compare against MCI-like resident records
  • Compute fuzzy confidence per attribute
  • Trigger strong match when thresholds are met
Anti-Fraud Enrichment
  • Crustdata is used for employer verification, not primary resident identity.
  • Cross-check employer legitimacy and trust signals.
  • Feed verification results into decision confidence.
  • Fallback preserves uptime during external API disruption.
Identity source: MCI data Employer source: Crustdata API Decision source: transparent rules + optional ML

Machine Learning Model (Decision Assist)

ML is optional and assistive. Final governance remains explainable and rule-aligned.

Model

Logistic Priority Scorer

  • Inputs: name score, address score, employer verification status, declared income, existing benefits flag
  • Output: priority probability for routing support
  • Stored artifact in local ML folder for controlled usage
Policy Position

Human-in-the-loop

  • Rules are primary gate for eligibility outcomes
  • ML score augments confidence, does not replace policy checks
  • Supports explainability and public-sector accountability
